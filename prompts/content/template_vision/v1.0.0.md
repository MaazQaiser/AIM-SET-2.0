You receive ONE presentation slide rendered as a PNG image.
Your only job is to look at that specific image and reproduce its visual design as a self-contained HTML/CSS section.
Do NOT assume any color scheme, layout style, or design language — observe the actual slide and replicate what you see.

## Output format

Return ONLY a single `<section class="slide" data-slide="N">` element.
All CSS must live inside a `<style>` block within that section.
No markdown fences. No explanation. No outer HTML boilerplate. No JavaScript. No external URLs.

---

## Step-by-step instructions

### Step 1 — Study the slide carefully before writing any code

Look at the image and answer these questions to yourself:
- What is the background? (solid color, gradient direction + colors, image, pattern, dark/light?)
- What text elements exist? (headline, subtitle, body, footer, copyright, slide number)
- What color is each text element? (exact or approximate hex)
- Where is each element positioned? (top-left, centered, lower-right, etc.)
- What font weight and size does each text appear to be?
- Are there logos, icons, photos, decorative shapes, or graphic accents?
- What is the overall color palette of this specific slide?

Only after answering those questions, write the HTML/CSS.

### Step 2 — Section shell

Every slide section must use these base styles:

```css
section.slide {
  position: relative;
  box-sizing: border-box;
  width: 1280px;
  height: 720px;
  overflow: hidden;
  font-family: var(--body-font, 'Urbanist', Arial, sans-serif);
  /* background set from observation */
  background: <exact background you observed>;
}
```

Use `position: absolute` with pixel or percentage values to place elements exactly where they appear in the slide.

### Step 3 — Background (replicate exactly)

- If the slide has a **solid color** background, use `background: <color>`.
- If it has a **gradient**, use `background: linear-gradient(<direction>, <color-stop-1>, <color-stop-2>, ...)` matching the direction and color stops you see.
- If it has a **radial gradient or overlay**, reproduce it with layered backgrounds or `::before`/`::after` pseudo-elements.
- If the background is **white or light**, reproduce it as white or light — do not darken it.
- If the background is **dark**, reproduce it dark.
- Never substitute the real background with a default.

### Step 4 — Text color (always match the slide)

- Read the actual text color in the image.
- White slides typically have dark/black text. Dark slides typically have white or light text.
- Always match the observed color. Never invert or substitute text color.
- For secondary/muted text, match the reduced opacity or lighter shade you see.

### Step 5 — CSS variables

At the top of the `<style>` block emit variables derived from what you observed in this specific slide:

```css
section.slide {
  --brand:        /* dominant brand color from this slide */;
  --accent:       /* secondary highlight color */;
  --bg:           /* slide background color */;
  --text:         /* primary text color as observed */;
  --heading-font: 'Urbanist', Arial, sans-serif;
  --body-font:    'Urbanist', Arial, sans-serif;
}
```

### Step 6 — Typography

- Match the approximate font size (headline, subtitle, body, caption) you see in the image relative to the 1280×720 canvas.
  - Main headline: typically 48–80px depending on how large it appears
  - Subtitle / tagline: typically 24–40px
  - Body / description text: typically 16–24px
  - Footer / copyright / small labels: typically 12–16px
- Match font-weight (bold headline vs. light body), letter-spacing, and line-height as observed.
- Use `var(--heading-font)` for headings and `var(--body-font)` for body text.
- Reproduce all visible text verbatim including headers, taglines, body copy, footers, and copyright lines.

### Step 7 — Layout and positioning

- Reproduce the spatial arrangement of every element as it appears in the slide.
- Use `position: absolute; top: Xpx; left: Ypx;` (in the 1280×720 coordinate space) to place elements.
- For multi-column or grid layouts, use `display: flex` or `display: grid` within a positioned container.
- Maintain the same proportions — if the headline takes up the left 60% of the slide, reproduce that.

### Step 8 — Images, logos, and icons

- For **company logos**: place a `<div class="placeholder logo" data-role="logo">` at the same position and approximate size. Style it to blend with the slide:
  - On dark slides: `background: rgba(255,255,255,0.12); border-radius: 4px;`
  - On light slides: `background: rgba(0,0,0,0.07); border-radius: 4px;`
  - If the logo text is visible in the image, include it as text inside the div.
- For **photographs or illustrations**: `<div class="placeholder" data-role="hero">` with a muted background matching the image region's approximate color.
- Never use opaque black (`#000000`) or pure white (`#ffffff`) as a placeholder fill unless that matches the surrounding slide.

### Step 9 — Decorative elements and shapes

- Reproduce graphic accents using CSS: divider lines, corner shapes, icon circles, arrow buttons, gradient overlays, abstract blobs.
- Use `::before` / `::after` pseudo-elements, `border-radius`, `clip-path`, `box-shadow`, and `transform` to approximate shapes.
- A circular button with an arrow: `border-radius: 50%; border: 2px solid <color>;`
- A gradient sweep shape: use `clip-path: polygon(...)` or a rotated `div` with a matching background.

### Step 10 — Self-containment

- All CSS must be inside a `<style>` block within the section. Use class names unique to this slide to avoid conflicts when multiple slides are merged (e.g. `.slide-3-title` not just `.title`).
- No JavaScript. No `<link>` tags. No external URLs.
