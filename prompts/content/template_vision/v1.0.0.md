You receive ONE presentation slide rendered as a PNG image (1280×720 logical pixels).

**Primary goal:** Produce a reusable HTML/CSS template section where every piece of content (text, images) is a clearly named, replaceable slot — while the layout, colors, fonts, and decorative elements are locked in exactly as observed.

The template is used like a fill-in-the-blank form: a content agent will later replace slot values with real data without touching any CSS or layout.

## Output format

Return ONLY a single `<section class="slide template-slide" data-slide="N">` element.
All CSS must live inside a `<style>` block within that section.
No markdown fences. No explanation. No outer HTML boilerplate. No JavaScript. No external URLs.

---

## Phase 1 — Observe and map before coding (mandatory)

The canvas is **1280 px wide × 720 px tall**.

Before writing code, answer every question:

1. **Background** — exact color, gradient direction + stops, or image area?
2. **Every visible element** — estimate bounding box: top / left / width / height in px.
3. **Z-order** — background → shapes/decorations → images → text (always code in this order).
4. **Text inventory** — list every text block with its role:
   - What is it? (company name, headline, tagline, body paragraph, bullet point, label, footer, copyright, slide number)
   - What does it say? (copy verbatim — this becomes the placeholder default value)
   - What color is it?
   - What approximate font size? (use the 1280×720 scale: headline ~8–11% of 720px, body ~2.5–3%)
5. **Image / logo regions** — position and approximate size.
6. **Decorative shapes** — lines, circles, gradients, blobs, arrows, stripes.

---

## Phase 2 — Slot naming convention (critical for content generation)

Every piece of replaceable content gets a `data-slot` attribute with a **unique, descriptive name**.

Naming pattern: `slide-N-role` where N is the slide number.

| Content type | Example slot name |
|---|---|
| Company / brand name | `slide-1-company-name` |
| Main headline | `slide-1-headline` |
| Subtitle / tagline | `slide-1-tagline` |
| Body paragraph (first) | `slide-1-body-1` |
| Body paragraph (second) | `slide-1-body-2` |
| Bullet point list | `slide-1-bullets` |
| Footer / credit line | `slide-1-footer` |
| Copyright | `slide-1-copyright` |
| Slide number label | `slide-1-number` |
| Label / category tag | `slide-1-label-1`, `slide-1-label-2` |
| Hero / main image | `slide-1-hero-image` |
| Logo image | `slide-1-logo` |
| Supporting image | `slide-1-image-2` |
| Chart / data visual | `slide-1-chart` |
| CTA / button label | `slide-1-cta` |

Apply `data-slot` to the outermost element of each content block. For text spans inside a paragraph, apply it to the `<p>` or `<h1>` directly.

## Phase 3 — Section shell

```html
<section class="slide template-slide" data-slide="N">
  <style>
    section.slide[data-slide="N"] {
      --brand:        /* primary brand color */;
      --accent:       /* secondary highlight color */;
      --bg:           /* slide background */;
      --text:         /* primary text color */;
      --text-muted:   /* secondary/muted text color */;
      --heading-font: 'Urbanist', Arial, sans-serif;
      --body-font:    'Urbanist', Arial, sans-serif;

      position: relative;
      box-sizing: border-box;
      width: 1280px;
      height: 720px;
      overflow: hidden;
      font-family: var(--body-font);
      background: /* EXACT value from Phase 1 */;
    }
    /* ... rest of styles ... */
  </style>
  <!-- content elements here -->
</section>
```

Scope **all** CSS selectors to `section.slide[data-slide="N"]` or use `.sN-*` class names.
This prevents any style from leaking between slides when the full deck is rendered.

Every child element uses `position: absolute` anchored within the 1280×720 coordinate space.

## Phase 4 — Background (replicate exactly)

- Solid → `background: #hex`
- Linear gradient → `background: linear-gradient(Xdeg, #stop1 0%, #stop2 100%)` — exact direction and stops
- Radial / glow → `background: radial-gradient(...)`
- Multi-layer overlay → stacked `background` or `::before` pseudo-element
- **Never** substitute a different color or default to white/black

---

## Phase 5 — Contrast (always match the original)

- Dark background (luminance < 40%) → text **must** be light (`#fff` or near)
- Light background (luminance > 60%) → text **must** be dark (`#111` or near)
- Text over an image placeholder → add `background: rgba(0,0,0,0.45)` backdrop on the text container
- Never invert text color from what you observe

---

## Phase 6 — Overlap prevention (build in z-layers)

Code elements in this strict order: background → decorative shapes → image placeholders → text.

Rules:
1. Never place two text containers at the same `top` value.
2. If headline ends at `top + height`, next element starts at `top + height + gap` (minimum 16px gap).
3. Large display text (>60px) needs at least 24px vertical clearance from any other element.
4. Every text container must have an explicit `width` matching its observed column:
   - Full width: `width: 1100px; left: 90px`
   - Left column: `width: 560px; left: 90px`
   - Right column: `width: 500px; left: 700px`
5. Logo/image placeholder must not overlap any text block.
6. After positioning everything, do a mental collision check — fix any overlaps before outputting.

---

## Phase 7 — Typography (match the original precisely)

Estimate font size from Phase 1 proportions (fraction of 720px):

| Role | Fraction | px range |
|---|---|---|
| Hero / display title | 8–12% | 58–86px |
| Main headline | 5–8% | 36–58px |
| Subtitle / tagline | 3–5% | 22–36px |
| Body / description | 2–3% | 14–22px |
| Tag / label / small | 1.5–2% | 11–14px |
| Footer / copyright | 1–1.5% | 7–11px |

Match exactly:
- `font-weight`: 800 display, 700 bold, 600 semibold, 400 regular, 300 light
- `letter-spacing`: headlines −0.02em to −0.04em; uppercase labels 0.06em–0.12em
- `line-height`: headline stacks 1.0–1.1; body copy 1.5–1.6

**Text in elements**: use the actual text from the slide as the default slot value (not a generic placeholder). The original text shows the intended length and style — content generation will replace it.

## Phase 8 — Slot implementation for text

Each text element must be coded like this:

```html
<!-- headline example -->
<h1
  class="sN-headline"
  data-slot="slide-N-headline"
  data-slot-type="text"
  data-slot-label="Main headline"
>
  <Company Name>
</h1>

<!-- body paragraph example -->
<p
  class="sN-body-1"
  data-slot="slide-N-body-1"
  data-slot-type="text"
  data-slot-label="Description"
>
  Here will be the description of the specific company which will be mentioned in it.
</p>

<!-- tag/label list example — each tag is its own slot -->
<div class="sN-tags">
  <span class="sN-tag" data-slot="slide-N-label-1" data-slot-type="text" data-slot-label="Tag 1">Generative AI</span>
  <span class="sN-sep">·</span>
  <span class="sN-tag" data-slot="slide-N-label-2" data-slot-type="text" data-slot-label="Tag 2">Image recognition</span>
</div>
```

Required attributes on every content element:
- `data-slot="slide-N-role"` — unique slot id
- `data-slot-type="text"` or `"image"`
- `data-slot-label="Human readable name"` — shown to the content editor

## Phase 9 — Slot implementation for images and logos

```html
<!-- logo placeholder -->
<div
  class="sN-logo"
  data-slot="slide-N-logo"
  data-slot-type="image"
  data-slot-label="Company logo"
  style="position:absolute; top:Xpx; left:Ypx; width:Wpx; height:Hpx;
         background: rgba(255,255,255,0.10); border-radius:4px; display:flex;
         align-items:center; justify-content:center; overflow:hidden;"
>
  <!-- If logo text is visible in the slide, include it styled here -->
</div>

<!-- hero image placeholder -->
<div
  class="sN-hero"
  data-slot="slide-N-hero-image"
  data-slot-type="image"
  data-slot-label="Hero image"
  style="position:absolute; top:Xpx; left:Ypx; width:Wpx; height:Hpx;
         background: rgba(128,128,128,0.15); border-radius:Rpx; overflow:hidden;"
>
</div>
```

- Use `rgba` fills that blend with the slide — never opaque black or white
- Dimensions must match the observed region exactly
- Add `border-radius` only if the original had rounded corners

---

## Phase 10 — Decorative elements (lock these in — not replaceable slots)

Decorative elements are **not** slots. They are fixed CSS that defines the template's visual identity.

Reproduce every graphic accent:
- Thin lines → `<div>` with `height:1px` and observed color
- Gradient overlays / blobs → `::before` or `::after` with `clip-path` and matching gradient
- Color bands / stripes → positioned `div` with exact dimensions and color
- Bullet dots / separators → inline `<span>` elements, not slots
- Circular buttons (e.g. arrow-in-circle) → CSS `border-radius:50%; border: 2px solid <color>`

- [ ] Every text block has `data-slot`, `data-slot-type`, `data-slot-label`
- [ ] Every image/logo region has `data-slot`, `data-slot-type="image"`, `data-slot-label`
- [ ] Background exactly matches the original slide
- [ ] All text colors are correct for their background (contrast passes)
- [ ] No two text blocks overlap
- [ ] Logo/image placeholder does not overlap text
- [ ] Font sizes proportional to what is visible in the image
- [ ] All original text is included as default slot values
- [ ] All CSS scoped to `section.slide[data-slide="N"]` or `.sN-*` classes
